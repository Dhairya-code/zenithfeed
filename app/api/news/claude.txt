What about 3 layer architecture

09:41
Great thinking! You're right.

What is 3 Layer Architecture?
It means separating your app into 3 distinct layers:

Layer	Name	What It Does
Layer 1	Presentation Layer	What user sees — HTML, CSS, JS frontend
Layer 2	Business Logic Layer	The brain — Node.js backend, API calls, GPT processing
Layer 3	Data Layer	Storage — Database, cache, API responses
Current Problem With Our Code
Right now everything is in one HTML file — API keys are exposed in frontend. Anyone can open browser dev tools and steal your NewsAPI and OpenAI keys.

Judges will immediately notice this and question it.

Proper 3 Layer Structure for ZenithFeed
ZenithFeed/
│
├── frontend/              ← Layer 1 (Presentation)
│   ├── index.html
│   ├── style.css
│   └── script.js          (only calls YOUR backend, no API keys)
│
├── backend/               ← Layer 2 (Business Logic)
│   ├── server.js          (Node.js + Express)
│   ├── newsService.js     (NewsAPI calls)
│   ├── aiService.js       (GPT API calls)
│   └── .env               (API keys hidden here)
│
└── package.json           ← Layer 3 dependencies
How Data Flows
User clicks interest
       ↓
Frontend sends request to YOUR backend
       ↓
Backend calls NewsAPI → gets articles
       ↓
Backend calls GPT API → gets summary + credibility
       ↓
Backend sends clean data back to frontend
       ↓
Frontend displays news cards
Why This Matters for Judges
When judges ask "what is your architecture?" you say:

"ZenithFeed follows a 3 layer architecture. The Presentation Layer is our HTML CSS JS frontend. The Business Logic Layer is our Node.js Express backend which handles all NewsAPI and GPT API calls securely. The Data Layer manages API responses and will use MongoDB for caching in production. This ensures API keys are never exposed on the client side and the system is scalable."

That answer will genuinely impress them.

Option A — Keep current single file version Simpler, works for demo, but API keys exposed

Option B — Build proper 3 layer Node.js version More impressive, secure, judges will love it, but needs Node.js installed

import { NextRequest, NextResponse } from "next/server";
// ===== TYPES =====
type NewsArticle = {
  title: string;
  description: string;
  url: string;
  source: { name: string };
  publishedAt: string;
};
type AnalyzedArticle = NewsArticle & {
  credibility: "Verified" | "Unverified" | "Likely Fake";
  summary: string;
  reason: string;
};
// ===== FETCH NEWS FROM NEWSAPI =====
async function fetchArticles(topics: string[]): Promise<NewsArticle[]> {
  const query = topics.join(" OR ");
  const url = https://newsapi.org/v2/everything?q=${encodeURIComponent(
    query
  )}&language=en&pageSize=9&sortBy=publishedAt&apiKey=${process.env.NEWS_API_KEY};
  const res = await fetch(url);
  const data = await res.json();
  if (data.status !== "ok") {
    throw new Error(data.message || "NewsAPI error");
  }
  // Filter out removed articles
  return (data.articles || []).filter(
    (a: NewsArticle) => a.title && a.title !== "[Removed]"
  );
}
// ===== ANALYZE ARTICLE WITH GPT =====
async function analyzeArticle(
  title: string,
  description: string
): Promise<{ summary: string; credibility: string; reason: string }> {
  const prompt = You are a news credibility analyzer for Indian users.
Analyze this news article and return ONLY a valid JSON object. No extra text.
Title: "${title}"
Description: "${description || "No description available"}"
Return exactly this JSON:
{
  "summary": "2 line summary of the article in simple English",
  "credibility": "Verified" or "Unverified" or "Likely Fake",
  "reason": "One line reason for the credibility score"
};
  const res = await fetch("https://api.openai.com/v1/chat/completions", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: Bearer ${process.env.OPENAI_API_KEY},
    },
    body: JSON.stringify({
      model: "gpt-4o-mini",
      messages: [{ role: "user", content: prompt }],
      max_tokens: 200,
      temperature: 0.3,
    }),
  });
  const data = await res.json();
  const text = data.choices?.[0]?.message?.content || "";
  try {
    const cleaned = text.replace(/json|/g, "").trim();
    return JSON.parse(cleaned);
  } catch {
    // Fallback if GPT response fails
    return {
      summary: description || "Summary not available.",
      credibility: "Unverified",
      reason: "AI analysis unavailable for this article.",
    };
  }
}
// ===== MAIN POST HANDLER =====
export async function POST(req: NextRequest) {
  try {
    const body = await req.json();
    const topics: string[] = body.topics || ["technology"];
    // Step 1 — Fetch news articles
    const articles = await fetchArticles(topics);
    if (articles.length === 0) {
      return NextResponse.json({ articles: [] });
    }
    // Step 2 — Analyze each article with GPT
    const analyzedArticles: AnalyzedArticle[] = await Promise.all(
